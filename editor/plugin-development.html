插件开发指南#

本文档教你如何为 Infinite Map 编写自定义编辑器插件。

插件是什么#

插件是实现 InfiniteMapPlugin 接口的普通对象。每个插件负责一种编辑能力（如选择、拖拽、历史记录），通过统一的协议接入画布运行时。

核心价值：

  • 可组合：多个插件可以堆叠，composePlugins() 自动处理依赖和顺序
  • 可开关：通过 enabled 字段按需启用/禁用
  • 不侵入内核：所有编辑能力都实现在插件层，core 包保持不变

最小插件#

1import type { InfiniteMapPlugin } from "@qiuyulc/infinite-map";
2
3const myPlugin: InfiniteMapPlugin = {
4  id: "my-plugin",
5  setup: (ctx) => {
6    console.log("插件已挂载，当前节点数:", ctx.getNodes().length);
7  },
8  teardown: () => {
9    console.log("插件已卸载");
10  },
11};

这就是一个合法的最小插件。它只有生命周期钩子，不参与任何交互。

插件接口全览#

1type InfiniteMapPlugin = {
2  // ---- 元信息 ----
3  id: string;
4  enabled?: boolean;
5  priority?: number;
6  provides?: string[];
7  requires?: string[];
8  order?: { before?: string[]; after?: string[] };
9
10  // ---- 生命周期 ----
11  setup?: (ctx: MapContext) => void;
12  teardown?: () => void;
13
14  // ---- 非指针输入 ----
15  input?: InputHandlers;
16
17  // ---- 指针输入管线 ----
18  hitTests?: HitTestContributor[];
19  pointerDownProcessors?: PointerDownProcessor[];
20  gestures?: Gesture[];
21  inputHooks?: InputPipelineHooks;
22
23  // ---- 视觉输出 ----
24  overlay?: OverlayComponent;
25  slot?: "background" | "overlay" | "hud";
26  overlayPointerEvents?: "none" | "auto";
27
28  // ---- 命令 ----
29  commands?: Record<string, Command>;
30};

下面按功能域逐一讲解。

1. 元信息与依赖声明#

1{
2  id: 'my-drag',
3  priority: 100,       // 数字越大越靠前
4  provides: ['drag'],  // 声明自己提供的能力
5  requires: ['selection', 'commands'],  // 声明依赖的能力或插件
6  order: { before: ['resize'], after: ['selection'] },  // 显式排序
7}
  • provides：默认包含自身 id，可以扩展声明额外的能力名
  • requires：被依赖的插件必须存在（可以是插件 id 或能力名），否则 composePlugins() 抛错
  • order：显式指定前后关系，优先级低于 requires（依赖关系最优先）
  • priority：同层级内排序，数字大的在前

2. 生命周期#

1{
2  setup: (ctx) => {
3    // 插件挂载时调用一次
4    // 在这里：初始化 store、注册 service、订阅事件总线
5    const unsub = ctx.bus.on('selection:change', ({ ids }) => {
6      console.log('选中变化:', ids);
7    });
8  },
9  teardown: () => {
10    // 插件卸载时调用一次
11    // 在这里：解绑事件、清理 store、释放资源
12  },
13}

setup 中可以安全地：

  • 读写 ctx.store
  • 注册 ctx.registerService(name, service)
  • 订阅 ctx.bus.on(event, handler)
  • 读 ctx.getNodes()、ctx.getCamera() 等

注意：setup 中不应调用 ctx.applyPatches() —— 那是命令/手势的责任。

3. 非指针输入#

1{
2  input: {
3    onWheel: (e, ctx) => {
4      // 处理滚轮事件
5      return { handled: true };  // 阻止默认行为
6    },
7    onKeyDown: (e, ctx) => {
8      if (e.code === 'Space') {
9        ctx.store.set('space:pressed', true);
10        return { handled: true };
11      }
12      return { handled: false };
13    },
14    onKeyUp: (e, ctx) => {
15      if (e.code === 'Space') {
16        ctx.store.set('space:pressed', false);
17        return { handled: true };
18      }
19      return { handled: false };
20    },
21    onContextMenu: (e, ctx, hit) => {
22      // 处理右键菜单
23      return { handled: false };
24    },
25  },
26}

返回值 { handled: true } 会阻止事件的默认行为和后续处理。

4. 指针输入管线（核心）#

指针（鼠标/触控）输入走一个统一的管线：

Raw Pointer Event → hitTests（命中检测）→ 确定指针命中了什么 → pointerDownProcessors（非互斥预处理）→ selection 等逻辑 → gestures（互斥手势）→ drag / resize / rotate / marquee / pan

4.1 HitTestContributor（命中检测）#

1{
2  hitTests: [
3    {
4      id: 'node-hit',
5      priority: 100,
6      hitTest: (e, ctx, info) => {
7        // info.kind === 'pointer' | 'contextmenu'
8        const node = ctx.queryNodesInWorldRect({
9          x: e.world.x, y: e.world.y, w: 1, h: 1,
10        })[0];
11        if (node) {
12          return { kind: 'node', id: node.id };
13        }
14        return null;  // 不处理，交给下一个 contributor
15      },
16    },
17  ],
18}

返回 null 表示"不处理"，由下一个 contributor 竞争。多个 contributor 按 priority 降序执行，第一个非 null 结果获胜。

4.2 PointerDownProcessor（非互斥预处理）#

1{
2  pointerDownProcessors: [
3    {
4      id: 'selection-processor',
5      priority: 50,
6      onPointerDown: (e, ctx, hit) => {
7        if (hit.kind === 'node') {
8          ctx.store.set('selection:ids', [hit.id]);
9          ctx.requestRender();
10        }
11        // 不返回任何值 = 不影响后续 gesture
12        // 返回 { stop: true } = 阻止后续 gesture 启动
13        // 返回 { hit: newHit } = 修改本次有效命中
14      },
15    },
16  ],
17}

4.3 Gesture（互斥手势）#

最复杂的插件类型。一个 Gesture 有五个回调：

1{
2  gestures: [
3    {
4      id: 'my-drag',
5      priority: 100,
6      canStart: (e, ctx, hit) => {
7        // 判断是否可以启动此手势
8        // 返回 false 则跳过，交给下一个 gesture
9        return e.button === 0 && hit.kind === 'node';
10      },
11      onStart: (e, ctx, hit) => {
12        // 手势开始：保存初始状态
13        ctx.store.set('my-drag:state', {
14          id: hit.kind === 'node' ? hit.id : '',
15          startX: e.world.x,
16          startY: e.world.y,
17        });
18      },
19      onMove: (e, ctx) => {
20        // 每帧移动：计算偏移，更新状态
21        const state = ctx.store.get('my-drag:state');
22        if (!state) return;
23        // ...
24        ctx.requestRender();
25      },
26      onEnd: (e, ctx) => {
27        // 手势结束：提交最终变更
28        const state = ctx.store.get('my-drag:state');
29        if (!state) return;
30        ctx.applyPatches(
31          [{ type: 'move', id: state.id, x: finalX, y: finalY }],
32          { source: 'plugin', plugin: 'my-drag', reason: 'drag', phase: 'end' },
33        );
34        ctx.store.set('my-drag:state', null);
35        ctx.requestRender();
36      },
37      onCancel: (e, ctx) => {
38        // 手势被取消（如按 Escape、切换窗口）
39        ctx.store.set('my-drag:state', null);
40        ctx.requestRender();
41      },
42    },
43  ],
44}

关键约定：

  • onMove 期间建议用 phase: 'move' 提交 patch（走 DOM 直写性能路径）
  • onEnd 时用 phase: 'end' 提交最终 patch
  • 不同 phase 的 patch 会被 history 插件自动合并为一条 undo 记录

4.4 InputPipelineHooks（输入管线钩子）#

1{
2  inputHooks: {
3    onBeforeHitTest: (e, ctx, info) => {
4      // 在 hitTest 之前执行（用于记录鼠标位置等）
5    },
6    onAfterHitTest: (hit, e, ctx, info) => {
7      // 在 hitTest 之后执行（用于 hover 检测等）
8    },
9    onHoverChange: ({ prev, next, e }, ctx) => {
10      // hover 目标变化时触发
11    },
12    onBeforeGesture: ({ phase, gestureId, hit, e }, ctx) => {
13      // 手势启动前
14    },
15    onAfterGesture: ({ phase, gestureId, hit, e }, ctx) => {
16      // 手势结束后
17    },
18  },
19}

5. 视觉输出：Overlay#

1import type { OverlayComponent } from "@qiuyulc/infinite-map";
2
3const MyOverlay: OverlayComponent = ({ ctx }) => {
4  const camera = ctx.getCamera();
5  return (
6    <div style={{ position: "absolute", left: 0, top: 0 }}>
7      Zoom: {camera.zoom.toFixed(2)}
8    </div>
9  );
10};
11
12const myPlugin: InfiniteMapPlugin = {
13  id: "my-info-panel",
14  slot: "hud",
15  overlay: MyOverlay,
16};

slot 决定了渲染层级：

  • 'background'：节点层之下（用于背景效果）
  • 'overlay'：节点层之上（用于选择框/对齐线等编辑辅助）
  • 'hud'：最上层（用于面板/工具栏/小地图等 UI）

overlayPointerEvents: 'auto' 时 overlay 可以接收鼠标事件（如 resize handle 需要点击）。

6. 命令#

1{
2  commands: {
3    'my-plugin.doSomething': {
4      id: 'my-plugin.doSomething',
5      title: 'Do Something',
6      shortcut: 'Mod+K',  // 可选：默认快捷键
7      run: (ctx, payload) => {
8        // 执行命令逻辑
9        ctx.applyPatches(
10          [{ type: 'add', node: { id: 'new', x: 0, y: 0, width: 100, height: 100 } }],
11          { source: 'plugin', plugin: 'my-plugin', reason: 'import' },
12        );
13      },
14    },
15  },
16}

命令可以被多种方式触发：

  • 快捷键（由 shortcuts 插件映射）
  • 工具栏按钮（通过 hud service 注册）
  • 右键菜单（通过 context menu service）
  • 程序调用：ctx.runCommand('my-plugin.doSomething')
  • API：apiRef.current.runCommand('my-plugin.doSomething')

7. Service 模式（跨插件协作）#

一个插件可以向 ctx.services 注册服务，供其他插件读取：

1// provider 插件
2{
3  id: 'my-service-provider',
4  provides: ['my-service'],
5  setup: (ctx) => {
6    ctx.registerService('my-service', {
7      getData: () => someSharedState,
8      doWork: (arg: string) => { /* ... */ },
9    });
10  },
11}
12
13// consumer 插件
14{
15  id: 'my-service-consumer',
16  requires: ['my-service'],
17  setup: (ctx) => {
18    const svc = ctx.getService<{ getData: () => unknown }>('my-service');
19    if (svc) {
20      console.log(svc.getData());
21    }
22  },
23}

内置 service 示例：

  • 'camera'：相机驱动服务（core 层注册）
  • 'document'：统一 patches 入口
  • 'hud'：Toolbar / ContextMenu 的贡献注册表
  • 'group'：编组操作（expandIds 等）
  • 'selection'：选择操作
  • 'dom-nodes'：DOM 节点引用（用于 Engine 模式直写）
  • 'engine'：引擎内部 store 和 cameraRef

8. Store 约定（跨插件共享状态）#

使用 STORE_KEYS 常量避免散落的字符串字面量：

1import { STORE_KEYS } from "@qiuyulc/infinite-map";
2
3// 读取选中
4const ids = ctx.store.get<string[]>(STORE_KEYS.selectionIds);
5
6// 写入吸附引导线
7ctx.store.set(STORE_KEYS.snapGuides, [{ axis: "x", value: 100 }]);
8
9// 订阅变化
10ctx.store.subscribe(STORE_KEYS.snapGuides, () => {
11  ctx.requestRender();
12});

关键 Store Key：

Key 用途
selection:ids 当前选中的节点 ID 列表
keyboard:space Space 键是否按下
drag:state 拖拽手势状态
resize:state 缩放手势状态
snap:config 吸附配置
snap:guides 当前吸附引导线
history:undoStack 撤销栈
history:redoStack 重做栈
clipboard:data 剪贴板数据
toolbar:items 工具栏贡献项
contextmenu:items 右键菜单贡献项

完整列表见 packages/infinite-map/src/editor/keys.ts。

9. 事件总线#

1// 发布
2ctx.bus.emit("selection:change", { ids: ["1", "2"] });
3
4// 订阅
5const unsub = ctx.bus.on("selection:change", ({ ids }) => {
6  console.log("选中变化:", ids);
7});
8// 卸载时解绑
9// unsub();

主要事件：

事件 用途
selection:change 选中变化
drag:start / drag:move / drag:end 拖拽生命周期
hover:change hover 目标变化
patches:applied patches 已应用（history 监听此事件）
command:run 命令已执行
camera:set / camera:changed 相机变更
history:undo / history:redo 撤销/重做
export:png 导出 PNG 请求

10. 完整插件模板#

1import type {
2  InfiniteMapPlugin,
3  MapContext,
4  Gesture,
5  HitTestContributor,
6  PointerDownProcessor,
7  Command,
8} from "@qiuyulc/infinite-map";
9import { STORE_KEYS } from "@qiuyulc/infinite-map";
10
11// ---- 类型 ----
12type MyPluginOptions = {
13  threshold?: number;
14};
15
16type MyGestureState = {
17  pointerId: number;
18  primaryId: string;
19  startWorld: { x: number; y: number };
20};
21
22// ---- 工厂函数 ----
23export function createMyPlugin(opts: MyPluginOptions = {}): InfiniteMapPlugin {
24  const threshold = opts.threshold ?? 10;
25
26  return {
27    // 元信息
28    id: "my-plugin",
29    priority: 50,
30    provides: ["myCapability"],
31    requires: ["commands", "selection"],
32    order: { after: ["selection"], before: ["drag"] },
33
34    // 生命周期
35    setup: (ctx) => {
36      ctx.store.set("my:data", { initialized: true });
37
38      ctx.bus.on("selection:change", ({ ids }) => {
39        if (ids.length > 0) {
40          ctx.store.set("my:lastSelection", ids);
41        }
42      });
43    },
44    teardown: () => {
45      // 清理（ctx.store 的 listener 由 core 自动清理）
46    },
47
48    // 非指针输入
49    input: {
50      onKeyDown: (e, ctx) => {
51        if (e.code === "KeyM" && e.modifiers.ctrl) {
52          ctx.runCommand?.("my-plugin.action");
53          return { handled: true };
54        }
55        return { handled: false };
56      },
57    },
58
59    // 命中检测
60    hitTests: [
61      {
62        id: "my-hit",
63        priority: 100,
64        hitTest: (e, ctx) => {
65          const nodes = ctx.queryNodesInWorldRect({
66            x: e.world.x - 5,
67            y: e.world.y - 5,
68            w: 10,
69            h: 10,
70          });
71          if (nodes.length > 0) {
72            return {
73              kind: "handle",
74              owner: nodes[0].id,
75              id: "my-handle",
76              handle: "my",
77            };
78          }
79          return null;
80        },
81      } satisfies HitTestContributor,
82    ],
83
84    // 指针预处理
85    pointerDownProcessors: [
86      {
87        id: "my-processor",
88        priority: 10,
89        onPointerDown: (e, ctx, hit) => {
90          if (hit.kind === "handle" && hit.handle === "my") {
91            // 做一些事，不阻断手势
92          }
93        },
94      } satisfies PointerDownProcessor,
95    ],
96
97    // 手势
98    gestures: [
99      {
100        id: "my-gesture",
101        priority: 100,
102        canStart: (e, ctx, hit) => {
103          return e.button === 0 && hit.kind === "handle" && hit.handle === "my";
104        },
105        onStart: (e, ctx, hit) => {
106          const st: MyGestureState = {
107            pointerId: e.pointerId,
108            primaryId: hit.kind === "handle" ? hit.owner : "",
109            startWorld: { x: e.world.x, y: e.world.y },
110          };
111          ctx.store.set("my:gestureState", st);
112        },
113        onMove: (e, ctx) => {
114          const st = ctx.store.get<MyGestureState>("my:gestureState");
115          if (!st || st.pointerId !== e.pointerId) return;
116          // 计算并提交 patch
117          ctx.requestRender();
118        },
119        onEnd: (e, ctx) => {
120          const st = ctx.store.get<MyGestureState>("my:gestureState");
121          if (!st || st.pointerId !== e.pointerId) return;
122          ctx.applyPatches([], {
123            source: "plugin",
124            plugin: "my-plugin",
125            reason: "drag",
126            phase: "end",
127          });
128          ctx.store.set("my:gestureState", null);
129          ctx.requestRender();
130        },
131        onCancel: (e, ctx) => {
132          ctx.store.set("my:gestureState", null);
133          ctx.requestRender();
134        },
135      } satisfies Gesture,
136    ],
137
138    // 输入钩子
139    inputHooks: {
140      onBeforeHitTest: (e, ctx) => {
141        ctx.store.set("my:lastPointerScreen", e.screen);
142      },
143    },
144
145    // 命令
146    commands: {
147      "my-plugin.action": {
148        id: "my-plugin.action",
149        title: "My Action",
150        shortcut: "Mod+M",
151        run: (ctx, payload) => {
152          const source = payload?.source ?? "api";
153          ctx.applyPatches([], {
154            source: "plugin",
155            plugin: "my-plugin",
156            reason: "keyboard",
157          });
158        },
159      } satisfies Command,
160    },
161  };
162}

11. 最佳实践#

  1. 一个插件一个职责。不要把 drag 和 clipboard 写在一个插件里。

  2. 用 STORE_KEYS 而不是字符串字面量。避免拼写错误，方便 IDE 跳转。

  3. Gesture 中 onMove 不要调 applyPatches 传 phase: 'start' 或 phase: 'end'。全程用 phase: 'move'，让 history 插件正确合并。

  4. 声明 requires 而非依赖隐式顺序。requires 会被 composePlugins 校验，顺序依赖不会被校验。

  5. 提供 TypeScript 类型导出。每个插件工厂的 Options 类型应该从包入口导出，方便使用者配置。

  6. teardown 中清理 ctx.store 写入的状态。避免热更新时残留脏数据。

  7. requestRender() 仅在需要重绘 overlay 时调用。不要在每帧都调——仅在状态变化后调一次。
